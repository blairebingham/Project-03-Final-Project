# Final Project Retrospective

### What Went Well?

- **Workflow:** Using VS Code and the Live Server extension made real-time updates very easy.

- **Documentation:** Having a `scope.md` and `plan.md` from the start prevented distractions and kept the project small.

- **Deployment:** Github Pages provided a fast, seamless way to go from local code to a live URL.

### What Went Wrong?

- **Image Management:** Initially, I did not have the image file linked correctly, which led to a broken link I had to go back and figure out how to fix.

- **Git Syncing:** Encountered a minor "merge conflict" when editing files directly on Github and locally at the same time.

- **Time Estimation:** Some tasks, like table styling, took longer than planned in the `plan.md` file.

### What I'd Do Differently

- **Structured Checkpoints:** I would implement more regular, structured check-ins to ensure my images, coding, etc. are functioning properly before committing and pushing to Github.

- **Time Management:** I would more adequately map out the expected time constraints of each task to more efficiently manage my time.
