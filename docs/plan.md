# Project Plan- Final Project (The Coffee Nook)
This document outlines the step-by-step execution strategy for developing and deploying the website.
## Phase 1 - Preparation & Setup
- **Environment Setup:** Initialize the project folder in VS Code.
- **Git Initialization:** Initialize a local Git repository.
- **Documentation:** Create `README.md`, `scope.md`, and `plan.md` to define the project structure.
- **Asset Gathering:** Select placeholder image for the cafe. 
## Phase 2 - Core Development
- **Document Structure:** Set up the basic HTML structure for `index.html` and `about.html`.
- **Navigation:** Build the shared `<nav>` component 
- **Content Injection:** 
    - Add the "hero" section and Specials list to `index.html`.
    - Add the "Our Story" text and the "Hours" table to `about.html`.
- **Semantic Check:** Ensure all content is wrapped in appropriate tage (`<main>`,`<section>`,`<footer>`).
## Phase 3 - Visual Design (CSS)
- **External Link:** Link `style.css` to both HTML files.
- **Base Styles:** Set the background colors, typography, and primary text colors.
- **Layout:** Use margins and padding to create room around text. 
- **Components:**
    - Style the navigation bar to look like a clean menu.
    - Format the hours table for better readability.
## Phase 4 - Deployment 
- **Final Review:** Proofread text and check for broken image links.
- **Push to Github:** Commit all changes and push the local repository to Github.
- **Activate Hosting:** Enable **Github Pages** in the repository settings.
- **Live Test:** Visit the live URL to ensure all styles and links work in a production environment. 
## Post-Launch
- Share the link for feedback.
- Document project reflection in `retrospective.md`.
- Submit project by 11:59 PM on May 8, 2026.
## Risk Analysis (TAME Method)

| Risk Event | Impact | Strategy | Action Plan |
| :--- | :--- | :--- | :--- |
| Broken Links | High | **Mitigate** | Test all href paths locally before pushing. |
| Scope Creep | Medium | **Avoid** | Stick strictly to this scope document. |
| Data Loss | High | **Mitigate** | Frequent Git commits and pushes. |
| Copyright | Medium | **Avoid** | Use only royalty-free images. |
